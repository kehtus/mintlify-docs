---
title: "Правила протокола"
description: "Правила funded-аккаунта, которые enforce'ит протокол on-chain"
---

Протокол сам следит за всеми правилами funded-аккаунта. Билдер не может смягчить или обойти их — только выбирает параметры в разрешённом диапазоне.

## Жёсткие правила

Хардкод, общие для всех:

| Правило | Значение |
|---------|----------|
| Max leverage | 5x notional/equity. Пробой → закрытие позиций |
| Profit split | 80% трейдер / 13% LP / 7% протокол |
| Profitable days для вывода | 4 дня с ростом ≥ 1% каждый |
| Checkpoint interval | 1 день |
| Subscription period | 30 дней |

## Trailing drawdown (HWM-based)

FA использует trailing floor: HWM обновляется на checkpoint раз в сутки, и прибыль трейдера фиксируется в floor по итогам дня. Внутри дня floor не подскакивает за PnL — но на следующем checkpoint поднимается, если день закрылся в плюс.

```
accountSize = $10k, drawdown = 10%
Старт:    HWM=$10k, floor=$9k
+$5k (checkpoint): HWM=$15k, floor=$13.5k
−$2k →    balance $13k < $13.5k → close
```

После вывода профита HWM сбрасывается на accountSize.

## Daily drawdown (опциональный)

Включается билдером при активации. Дневной лимит = 50% от trailing drawdown.

| Trailing | Daily |
|----------|-------|
| 5% | 2.5% |
| 10% | 5% |
| 25% | 12.5% |

Проверяется в каждом checkpoint и permissionless через `initiateClose()`. Скидка 20% на activation fee при включении.

## Whitelist перпов

Торговать можно только ликвидные перпы из whitelist протокола. Keeper мониторит позиции off-chain и при детекте unauthorized asset вызывает `initiateClose()`. Контракт сам верифицирует нарушение и закрывает аккаунт, если оно действительно было.

## Что это значит для билдера

Свои дополнительные правила (профит-таргет, минимум торговых дней, более жёсткий daily limit) билдер делает off-chain — это часть его челленджа и evaluation. Когда трейдер прошёл по критериям билдера, билдер открывает funded-аккаунт через SDK, и дальше протокол защищает капитал по своим правилам.
