> Builder 负责产品。Proportion 提供 funded layer。

对 Builder 来说，funded account 是 prop firm 中最消耗资本、也最重基础设施的一部分。Proportion 将这一层作为基础设施提供：资本、链上风控执行和结算。Challenge、evaluation、品牌、营销和 trader selection 仍然是 Builder 自己的产品。

Proportion 与 Hyperliquid Builder Codes API 兼容，因此 Builder 可以用较少的客户端改动，将其集成到现有 trading frontend 中。Builder 从协议购买资本访问权，并用 challenge revenue 或其他 selection product 的收入支付成本。

成为 Builder 需要向 LP vault 存入资金。该 deposit 作为 LP capital 参与收益，因此激活 FA 的有效成本低于名义金额。同一笔 deposit 也让 Builder 有 skin in the game：activation fee 旨在补偿预期 drawdown risk，但实际亏损仍可能降低 pool yield 和 `sharePrice`。

## Proportion 给 Builder 什么

Funded account 是 Proportion 提供的基础 primitive：一次 SDK call 创建一个隔离的链上账户，带有资本、drawdown limits 和自动风控执行。FA smart contract 在协议规则内属于用户。

| Builder layer — product | Protocol layer |
|-------------------------|----------------|
| Challenge 和 evaluation | 来自 LP vault 的资本 |
| Trader selection 和 onboarding | 链上 risk engine（drawdown、leverage、allowed perps） |
| 品牌、营销、定价 | Settlement：fees、subscriptions、profit split 80/13/7 |

边界：FA activation 之前由 Builder 处理；activation 之后由协议执行规则。

### Builder 同时也是 LP

成为 Builder 需要向 LP vault 存入至少 $10,000 USDC。这笔 deposit：

- 提供 funding quota：10x leverage（$10k → 最高 $100k active trader capital）
- 让 Builder 成为 LP pool 的一部分 — shares 参与整个协议所有 FA 的收益和损失，而不仅是自己的 FA

LP pool 从 activation fees 和 trader profits 中获得收益。触及限制的 trader drawdown 会降低 pool yield。Builder 同时在两个层面有 skin in the game：自己的产品，以及整个生态的健康。
